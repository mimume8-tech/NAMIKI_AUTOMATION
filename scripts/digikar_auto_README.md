# デジカル カルテ下書き一括保存スクリプト

受付一覧の全患者カルテに対して「主訴・所見の複写」「処置・行為の複写」「￥ボタンで下書き保存」を自動実行するスクリプトです。

## 前提条件

- Windows 11
- Google Chrome がインストール済み
- Node.js v18 以上がインストール済み

## セットアップ手順

### 1. Node.js のインストール（未導入の場合）

https://nodejs.org/ から LTS 版をダウンロードしてインストールしてください。

### 2. 依存パッケージのインストール

プロジェクトのルートフォルダで以下を実行します:

```bash
cd C:\NAMIKI_AUTOMATION
npm install
```

Playwright のブラウザは Chrome を直接使用するため、追加インストールは不要です。

### 3. 環境設定（任意）

ログイン情報を `.env` ファイルで管理する場合:

```bash
cp .env.example .env
```

`.env` を開いて、デジカルのログイン情報を入力してください。
（現行版はブラウザの既存ログインセッションを利用するため、`.env` は必須ではありません）

## 実行方法

### 手順

1. **Chrome を完全に閉じる**（タスクトレイのアイコンも右クリック→終了）

2. **スクリプトを実行**:
   ```bash
   node scripts/digikar_auto.js
   ```

3. **Chrome が自動起動**します。以下の手動操作を行ってください:
   - 証明書ダイアログが出たら **OK** をクリック
   - ログインページが表示されたら **ログイン** する

4. **受付画面が表示されたら**、ターミナルに戻って **Enter** を押す

5. 自動処理が開始されます。処理状況はターミナルに表示されます。

### 途中再開

スクリプトは処理済み患者を `scripts/processed.json` に記録します。
途中で中断しても、再実行時に処理済みの患者はスキップされます。

最初からやり直したい場合は `processed.json` を削除してください:

```bash
del scripts\processed.json
```

## 処理内容

各患者に対して以下を実行します:

1. カルテページへ遷移
2. 保険タブが複数ある場合（例:「精神通」「生活保」）は全タブを処理
3. 各タブで:
   - 「主訴・所見」の複写ボタンをクリック
   - 「処置・行為」の複写ボタンをクリック
   - 「￥」ボタンで下書き保存
4. 次の患者へ進む

## トラブルシューティング

### Chrome に接続できません
- Chrome が完全に終了していることを確認してください
- タスクマネージャーで `chrome.exe` が残っていないか確認

### 患者リンクが見つかりません
- 受付画面に患者が表示されていることを確認
- 正しい日付の受付画面が表示されているか確認

### 複写ボタンが見つかりません
- デジカルの画面構成が変更された可能性があります
- スクリプトのセレクター部分の調整が必要な場合があります

## ファイル構成

```
scripts/
  digikar_auto.js          # メインスクリプト
  digikar_auto_README.md   # この説明書
  processed.json           # 処理済み患者リスト（自動生成）
.env.example               # 環境変数テンプレート
```
